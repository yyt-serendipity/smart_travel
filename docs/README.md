# 文档索引

本文档用于说明 `docs/` 目录里每份文档的用途，以及推荐的阅读顺序。

## 推荐阅读顺序

1. `project-overview.md`
2. `Smart Travel 代码总览手册.md`
3. `Smart Travel 架构与部署说明.md`
4. `Smart Travel 数据库设计.md`
5. `Smart Travel 生产发布记录与 Runbook.md`
6. `Smart Travel 项目开发手册.md`

## 文档说明

### `project-overview.md`

适合第一次接手项目时快速了解：

- 项目定位
- 当前功能范围
- 前后台模块边界
- 当前代码与旧文档的主要差异

### `Smart Travel 代码总览手册.md`

适合改代码前查入口，重点关注：

- 各 app 职责
- 关键文件位置
- 典型请求链路
- 常见需求应该改哪里

### `Smart Travel 架构与部署说明.md`

适合部署、排障和讲系统结构时使用，重点包括：

- 架构分层
- 关键业务流
- 生产拓扑
- 环境变量
- 运维命令和常见故障排查

### `Smart Travel 数据库设计.md`

适合做数据库说明或答辩材料，重点包括：

- 业务表与框架支撑表
- ER 图
- 表名策略
- 关键字段与关系

### `Smart Travel 生产发布记录与 Runbook.md`

适合实际上线前后对照使用，重点包括：

- 最近一次真实发布记录
- 制品生成方式
- 覆盖部署步骤
- 验收结果与遗留事项

### `Smart Travel 项目开发手册.md`

适合开发协作时使用，重点包括：

- 本地开发流程
- 测试和构建要求
- 数据导入和文档同步要求
- 发布前后的检查项

## 当前项目事实

- 数据库物理表名已经去掉 `core_` 前缀。
- 社区发帖已改为轻量文本表单，不再使用富文本编辑器。
- AI 规划会自动保存登录用户自己的历史记录，并支持返回页恢复。
- 后台正式入口统一为 `/backoffice`，不再保留独立 Django Admin 登录入口。
